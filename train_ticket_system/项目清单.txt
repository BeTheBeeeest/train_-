运行说明
=====================================

环境要求:

- Windows操作系统

或使用命令行:
1. go run server/main.go

候补功能核心流程
=====================================

1. 查询车票 → 发现无票 → 提示候补选项
2. 输入乘客信息 → 创建候补订单 → 加入队列
3. 分配排队位置 → 模拟支付确认 → 订单生效
4. 等待兑现 → 查询位置状态 → 实时更新
5. 退票触发 → 自动分配 → 候补成功
6. 状态更新 → 队列前移 → 通知结果
